## COPYWRITING PRINCIPLES

### Vary Sentence Structure
**DON'T DO THIS (copy-paste risk):**
```
I've built X. I've achieved Y. I'm interested in Z.
I've built A. I've achieved B. I'm interested in C.
```

**DO THIS (natural variation):**
```
I architect X, which has achieved Y. This directly supports Z.
Building X is hard; I've solved it with Y, enabling Z.
X powers Z. I've built X to achieve Y.
```

All 18 versions in this package vary sentence structure significantly.

### Approach 1: Cold InMail to Lavinia (REJECTED)
**Why not:**
- InMail costs credits (5-15/month for Premium)
- Connection acceptance is MUCH higher than InMail open rate
- Connection = warmer path to building relationship
- InMail best for 3rd+ degree with lower connection likelihood (not Lavinia)

**When to use InMail instead:** If Lavinia declines connection after 2 weeks

### Approach 2: Direct Apply on Greenhouse Without Relationship (REJECTED)
**Why not:**
- No context in application stands out (resume-only)
- No warm introduction to recruiter (Ravi sees cold application)
- Post-application follow-up is much more effective with relationship
- Better to build relationship first, apply second

**Timeline correction:** Build relationship first (connection + follow-up), THEN apply, THEN post-app follow-up. This gives 2-3 relationship touches before + 1 after application.

### Approach 3: Comment-Only Strategy (REJECTED)
**Why not:**
- Comments build visibility but don't create obligation
- No direct relationship formed
- Lavinia + Ravi won't remember Adithya after scrolling feed
- Comments must be paired with connection request for maximum effect

**Best practice:** Comments → Connection → Follow-up (3-touch sequence)
